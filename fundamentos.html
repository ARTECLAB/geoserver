Linux — comandos que usaremos

GeoServer corre en Debian 13 dentro de VirtualBox. Estos comandos aparecen en las primeras clases. Si algunos son nuevos para ti, los iremos viendo juntos paso a paso.

Los comandos más frecuentes del curso
# Navegar el sistema de archivos pwd # ver en qué carpeta estás ls -la # listar archivos con detalles cd /opt/tomcat/webapps # entrar a una carpeta cd .. # subir un nivel # Trabajar con archivos cp geoserver.war /destino/ # copiar un archivo mv archivo /destino/ # mover o renombrar rm archivo # eliminar nano /etc/archivo.conf # editar un texto # Gestionar servicios con systemctl sudo systemctl start tomcat # iniciar Tomcat sudo systemctl stop tomcat # detener Tomcat sudo systemctl status tomcat # ver si está activo # Ver logs en tiempo real — el más útil del curso tail -f /var/log/tomcat11/catalina.out
⭐
El comando más importante del curso
tail -f /var/log/tomcat11/catalina.out muestra los logs de Tomcat en tiempo real. Cuando algo no funciona, este es el primer lugar donde mirar.
🌐
¿Cómo saber la IP de la VM?
Ejecutar ip a en la terminal de la VM. La IP de la interfaz de red (algo como 192.168.1.X o 10.0.2.X) es la que usas desde tu PC para acceder a GeoServer. El instructor configurará la red de VirtualBox en la primera clase.

Java y Tomcat — lo que necesitas saber

GeoServer es una aplicación Java que corre dentro de Tomcat. No es necesario saber programar en Java — en clase instalaremos todo juntos y explicaremos cada paso con calma.

⚠️
Dato clave para el curso
GeoServer 3 requiere Java 17 o 21 y Tomcat 11. La VM del curso viene con ambos preinstalados. Tomcat 9 y 10 no son compatibles con GeoServer 3.
☕
Java JDK 17
Motor que ejecuta GeoServer. La VM del curso ya viene con Java 17 o 21 instalado. Verificar con: java -version
🐈
Apache Tomcat 11
Servidor web Java. En Debian 13:
sudo apt install tomcat11
Sin descargas ni instaladores manuales.
📦
Archivo WAR
El instalador de GeoServer. Es un archivo .war que se copia en la carpeta webapps/ de Tomcat. Tomcat lo despliega solo.
🧠
Memoria Java (Heap)
GeoServer necesita memoria asignada con JAVA_OPTS="-Xms256m -Xmx1g". La VM del curso tiene solo 2GB de RAM en total — hay que dejar espacio para el sistema operativo y PostgreSQL, así que el heap se mantiene conservador.
Verificar la instalación
# Verificar Java java -version # Debe mostrar: openjdk version "17..." # Verificar Tomcat activo sudo systemctl status tomcat11 # Debe mostrar: active (running) # Abrir en el navegador de la VM http://192.168.X.X:8080 # Reemplaza X.X con la IP de tu VM (ver con: ip a)

Bloque 4

Arquitectura GeoServer — Workspace · Store · Layer

Todo en GeoServer sigue esta jerarquía sencilla. La veremos en detalle desde la primera clase con ejemplos prácticos.

📁
Workspace — el contenedor del proyecto
Agrupa todas las capas de un mismo proyecto. Aparece en la URL: /geoserver/bolivia_curso/wms
🗄️
Store — la fuente de datos
Apunta al dato original: una carpeta con Shapefiles o una base de datos PostGIS. Un Workspace puede tener varios Stores.
🗺️
Layer — la capa publicada
Lo que el cliente final consume via WMS o WFS. Cada capa tiene un estilo SLD asignado. Se identifica como workspace:nombre
🎨
Style (SLD) — el estilo visual
Define colores, bordes y etiquetas de la capa. GeoServer también acepta CSS, que es más fácil de escribir. Lo vemos en la Clase 5.
⭐
La URL lo dice todo
http://servidor:8080/geoserver/bolivia_curso/wms?LAYERS=bolivia_curso:departamentos
workspace = bolivia_curso  ·  capa = departamentos
Bloque 5

Conceptos GIS del curso

Estos términos aparecen desde la primera clase. Si son nuevos para ti, los explicaremos cuando los usemos.

📍
Coordenadas — EPSG:4326
Latitud: norte/sur (−90 a 90). Longitud: este/oeste (−180 a 180). La Paz: lat −16.50, lon −68.15. El sistema EPSG:4326 es el estándar WGS 84.
📐
Vectorial vs Raster
Vectorial: puntos, líneas y polígonos con atributos. Usamos Shapefiles y PostGIS. Raster: cuadrícula de píxeles (imágenes, elevación) — el curso es 100% vectorial, la VM no tiene RAM suficiente para publicar raster de forma confiable.
📁
Shapefile
El formato vectorial del curso. Siempre viene en varios archivos juntos: .shp, .dbf, .shx y .prj. Deben estar todos en la misma carpeta.
🗄️
PostGIS
Extensión espacial de PostgreSQL. Permite guardar y consultar geometrías (puntos, líneas, polígonos) en una base de datos relacional. Lo instalamos en la VM en la Clase 3.
🔍
BBox — Bounding Box
El rectángulo que envuelve una capa o zona del mapa. Formato: minLon, minLat, maxLon, maxLat. GeoServer lo usa en peticiones WMS para definir el área de la imagen.
🔢
CQL Filter
Filtro de GeoServer para mostrar solo parte de una capa. Ejemplo: departamento='LA PAZ' muestra solo ese departamento en el WMS.
🌐
