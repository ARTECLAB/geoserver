Linux — comandos básicos

GeoServer corre en un servidor Linux. No necesitas ser administrador de sistemas, pero sí moverte con soltura en la terminal. Estos comandos aparecen en las primeras dos clases.

Comandos que usarás en el curso
# Navegar el sistema de archivos pwd # mostrar dónde estás ls -la # listar archivos con detalles cd /opt/tomcat/webapps # ir a una carpeta cd .. # subir un nivel # Trabajar con archivos cp archivo.war /destino/ # copiar mv archivo.war /destino/ # mover o renombrar rm archivo.war # eliminar nano /etc/archivo.conf # editar un archivo de texto # Permisos y propietario chmod +x script.sh # hacer ejecutable chown tomcat:tomcat /opt/geoserver/data_dir # Servicios con systemctl systemctl start tomcat # iniciar systemctl stop tomcat # detener systemctl status tomcat # ver estado systemctl enable tomcat # iniciar al boot # Ver logs en tiempo real tail -f /opt/tomcat/logs/catalina.out # Descargar archivos wget https://url/geoserver.war curl -O https://url/geoserver.war

Bloque 2

Java y Tomcat — lo que necesitas saber

GeoServer es una aplicación Java que corre dentro de Tomcat. No necesitas programar en Java, pero sí entender cómo se instala, configura y se le cambian los parámetros de memoria.

⚠️
Cambio crítico en GeoServer 3
GeoServer 3 requiere Java 17 (mínimo) y Tomcat 11. Tomcat 9 ya no funciona — migró de javax.servlet a jakarta.servlet. Si instalas la versión equivocada, GeoServer no arranca.
☕
JDK vs JRE
JRE: solo ejecuta Java. JDK: ejecuta y compila. Necesitamos el JDK 17. En Debian 13: apt install default-jdk instala Java 17 directamente desde los repositorios oficiales.
🐈
Apache Tomcat
Servidor web Java (contenedor de servlets). En Debian 13 se instala con apt install tomcat10 sin descargas manuales. GeoServer se despliega como un WAR en webapps/.
📦
Archivo WAR
Web Application Archive — un ZIP con toda la aplicación Java lista para desplegar. Copiar geoserver.war a webapps/ es suficiente para instalarlo.
🧠
Memoria Java (Heap)
GeoServer necesita memoria configurada con JAVA_OPTS="-Xms512m -Xmx2g". Sin esto, se quedará sin memoria con datasets grandes.
Verificar Java y Tomcat después de instalar
# Verificar versión de Java instalada java -version # Debe mostrar: openjdk version "17..." # Verificar variable JAVA_HOME echo $JAVA_HOME # Debe mostrar: /usr/lib/jvm/java-17-openjdk-amd64 (Debian 13) # Estructura de Tomcat (lo que verás en el curso) /opt/tomcat/ ├── bin/ # startup.sh, shutdown.sh ├── conf/ # server.xml, tomcat-users.xml ├── webapps/ # aquí va geoserver.war └── logs/ # catalina.out (los logs)

Estándares OGC — el lenguaje de los mapas web

OGC (Open Geospatial Consortium) define los protocolos que permiten que cualquier cliente GIS consuma datos de cualquier servidor. GeoServer los implementa todos.

Servicio Nombre completo ¿Qué devuelve? Cuándo usarlo
WMS Web Map Service Imagen PNG/JPEG del mapa renderizado Visualización rápida en mapas web
WFS Web Feature Service Datos vectoriales en GeoJSON o GML Descargar y analizar datos en el cliente
WMTS Web Map Tile Service Teselas pre-generadas en caché Mapas base de alta performance
WCS Web Coverage Service Datos raster crudos (DEM, satélite) Análisis de datos raster
🔗
Una petición WMS desde el navegador
Prueba esta URL (con tu GeoServer activo) para ver cómo funciona un GetCapabilities: http://localhost:8080/geoserver/wms?SERVICE=WMS&VERSION=1.3.0&REQUEST=GetCapabilities
Anatomía de una petición WMS GetMap
http://localhost:8080/geoserver/wms? SERVICE=WMS # protocolo &VERSION=1.3.0 # versión del estándar &REQUEST=GetMap # qué pedir &LAYERS=bolivia:departamentos # workspace:capa &CRS=EPSG:4326 # sistema de coordenadas &BBOX=-22.9,-69.6,-9.7,-57.5 # área del mapa &WIDTH=800&HEIGHT=600 # tamaño de la imagen &FORMAT=image/png # formato de salida &TRANSPARENT=true # fondo transparente

Bloque 4

Arquitectura de GeoServer — Workspace · Store · Layer

Todo en GeoServer sigue esta jerarquía. Entenderla desde el principio evita la confusión de las primeras clases.

📁
Workspace — el contenedor del proyecto
Como una carpeta. Agrupa stores y capas relacionadas. Aparece en la URL: /geoserver/bolivia/wms
🗄️
Store — la fuente de datos
Puede ser un archivo (Shapefile, GeoTIFF) o una base de datos (PostGIS). Un workspace puede tener múltiples stores.
🗺️
Layer — la capa publicable
Lo que el cliente final consume. Una capa tiene un estilo SLD asignado. Los clientes piden capas por nombre: bolivia:departamentos
🎨
Style (SLD) — el estilo de la capa
Styled Layer Descriptor — XML que define colores, bordes y etiquetas. GeoServer también acepta CSS más legible. Clase 5 del curso.
⭐
La URL lo dice todo
http://servidor:8080/geoserver/bolivia/wms?LAYERS=bolivia:departamentos
workspace = bolivia · capa = departamentos
Bloque 5

Conceptos GIS fundamentales

GeoServer trabaja con datos geoespaciales. Estos términos aparecen desde la primera clase.

🔢
EPSG:4326 — WGS 84
El sistema de coordenadas geográficas estándar. Latitud y longitud en grados decimales. La Paz: lat −16.50, lon −68.15.
🗺️
EPSG:3857 — Web Mercator
Sistema de proyección que usa Google Maps, OSM y la mayoría de mapas web. GeoServer puede reproyectar sobre la marcha.
📐
Vectorial vs Raster
Vectorial: puntos, líneas y polígonos con atributos. Shapefile y PostGIS son los formatos del curso. Raster: cuadrícula de píxeles. GeoTIFF, DEM.
📦
Shapefile
El formato del curso. Siempre viene en varios archivos juntos: .shp (geometría), .dbf (atributos), .shx (índice) y .prj (proyección). Deben estar todos en la misma carpeta.
🔍
BBox (Bounding Box)
El rectángulo que envuelve una capa o una zona del mapa. Formato: minLon, minLat, maxLon, maxLat. Se usa en peticiones WMS y WFS.
⚡
GeoWebCache
Cache de teselas integrado en GeoServer. Pre-genera y almacena las imágenes WMS para servirlas mucho más rápido como WMTS.

Antes de la Clase 1

Lista de verificación

Confirma que tienes todo listo antes de la primera sesión.

1
Máquina virtual con Debian 13 en VirtualBox
Es el entorno del curso. Mínimo 2 GB de RAM y 20 GB de disco asignados a la VM.
2
Java 17 JDK instalado
En Debian 13: sudo apt install default-jdk. Verificar con java -version. Sin Java 17, GeoServer 3 no arranca.
3
Apache Tomcat instalado desde los repositorios de Debian
sudo apt install tomcat10 — sin descargas manuales. No instalar Tomcat 9, es incompatible con GeoServer 3.
4
Archivo WAR de GeoServer 3 descargado
De geoserver.org/download. El instructor indicará la versión exacta. Solo el archivo .war — no el bin.
5
QGIS instalado en tu computadora local
Lo usaremos para probar las capas publicadas en GeoServer desde el cliente.
✓
Prueba de verificación
Si puedes abrir el navegador en tu servidor y ver http://localhost:8080 mostrando la página de bienvenida de Tomcat, estás listo para la Clase 2 donde instalaremos GeoServer sobre esa misma VM.
